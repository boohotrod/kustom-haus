# B-2.4 ‚Äî Module ‚Üî Field Namespace Binding & Ownership

**Status:** implemented (preview-only, additive). No real DB migration.

## What shipped

- **Schema (no migration):** `module_namespace_bindings`, `namespace_ownership_history` Drizzle definitions in `src/db/schema/registry/moduleNamespaces.ts`.
  - `binding_kind`: `owner | allowed | required | reserved`
  - `auto_grant_to_dependents boolean DEFAULT false` (decision #1 ‚Äî schema + flag only; runtime auto-grant lands in B-3).
  - `grant_reason` is `NOT NULL` on every binding; ownership transfers always logged with `change_reason`.
- **Rules helper** `src/lib/namespace-ownership.ts`:
  - `findOwner()` ‚Äî dot-prefix matching (`vehicle` covers `vehicle.engine.*`, decision #3).
  - `canModuleUseNamespace()` ‚Äî owner / allowed / required / parent_allowed / dependency (only when owner flagged auto-grant) / superadmin.
  - `checkOwnershipInvariant()` ‚Äî surfaces `no_owner` / `multiple_owners`.
  - `requireReason()` + `ChangeReasonRequiredError` ‚Äî applied to module state changes, field status transitions, binding grant/revoke.
  - `transitiveDeps()` helper.
- **Lifecycle hardening:**
  - `assertFieldTransition()` enforces non-empty `change_reason` on every field status change.
  - `/modules` state-change prompt now blocks empty reason and writes it into the audit payload.
  - Binding grant / revoke require reason via the same helper.
- **Seed bindings (decision #4):** every namespace referenced by any field has an owner ‚Äî the invariant is green at boot. See `SEED_NAMESPACE_BINDINGS`.
- **Reserved visibility (decision #2):** reserved bindings are only rendered for `builder_admin` / `superadmin`.

## UI

- `/modules` ‚Äî module key is now a link to the detail page.
- `/modules/$moduleKey` (new) ‚Äî header (category / phase / state / visibility), Direct + transitive dependencies card, Namespaces card (Owned / Required / Allowed / Reserved-admin-only) with add/revoke (reason required), bindings table.
- `/modules/$moduleKey/fields` (new) ‚Äî fields under namespaces this module owns or is allowed to use, with owner-module badge and access-decision badge.
- `/registry/fields` ‚Äî new **Owner module** column linking to the owner module; `no_owner` shown as destructive badge if invariant breaks.
- `/registry/fields/$fieldId` ‚Äî new Namespace-usage panel above the tabs (owner + consuming modules).
- `/registry/fields/lifecycle` ‚Äî reason now hard-required; user-visible alert on empty input.

## i18n

`modules.detail.*`, `modules.namespaces.*`, `modules.lifecycle.*`, `registry.fields.cols.ownerModule`, `registry.fields.namespaceUsage.*` added across `en` / `hu` / `no`.

## Out of scope (per directive)

AI Translation Cache, Workflow Engine, GDPR Engine, Backup Engine, Export Engine, BBS portal business logic, real DB migration ‚Äî none touched.
